### 1.BOM

```
浏览器对象模型，可以使我们通过JS来操作浏览器；
在BOM中为我们提供了一组对象，用来完成对浏览器的操作：
	Window:
		- 代表的是整个浏览器窗口，同时window也是网页中的全局对象
	Navigator:
		- 代表的是当前浏览器的信息，通过该对象可以来识别不同的浏览器
		- navigator常见对象属性：
			- userAgent：是一个字符串，记录的是浏览器的信息
	Location:
		- 代表当前浏览器的地址栏信息，通过该对象可以获取地址栏信息，或者操作浏览器跳转页面
		- location对象属性：
			- host：设置或返回主机名和当前url的端口号
			- href：设置或返回完整的url
			- port：设置或返回当前url的端口号
		- location对象方法：
			- assign()：加载新的文档
			- reload()：重新加载当前文档
			- replace()：用新的文档替换当前文档
	History:
		- 代表浏览器的历史记录可以通过该对象操作浏览器的历史记录
			由于隐私的原因，该对象不能获取到具体的历史记录，只能操作浏览器向前或向后翻页；
			而且该操作只在当此访问时有效
		- history对象属性：
			- lenth：返回浏览器历史列表中的url数量
	Screen:
		- 代表用户的屏幕信息，通过该对象可以获取到用户的屏幕的相关信息
		
这些BOM对象在浏览器中都是作为window对象的属性保存的，可以通过window对象来使用，也可以直接使用；
```

### 2.DOM

```
每一个载入浏览器的HTML文档都会成为Document对象；该对象使我们可以从脚本中对HTML页面中的所有元素进行访问；
Document对象是window对象的一部分；
Document对象的属性：
	- cookie：设置会返回与当前文档有关的所有cookie
	- domain：返回当前文档的域名
	- lastModified：返回文档最后修改的日期和时间
	- referrer：返回载入当前文档的文档的url
	- title：返回当前文档的标题
	- URL：返回当前文档的url
	
Document对象方法：
	- close()：关闭用document.open()方法打开的输出流，并显示选定的数据
	- getElementById()：返回对拥有指定id的第一个对象的引用
	- getElementByName()：返回带有指定名称的对象集合
	- getElementByTagName()：返回带有指定标签名的对象集合
	- open()：打开一个流，以收集来自任何document.write()或document.writeln()方法的输出
	- write()：向文档写HTML表达式或JavaScript代码
	- wrireln()：等同于write()方法，不同的是在每个表达式之后写一个换行符
```

### 3.你平时工作中都遇到了哪些坑？

```
	我记忆比较深的是一次debugger反调试，我们一般过debugger都是经过call stack调用栈，找到debugger关键函数，重写关键函数，但是我重写关键函数之后，刷新页面还是出现debugger，后来仔细研究后发现js文件名中有一个时间戳，后来是通过reres插件做中间人攻击解决的；
	还有一次就是一个url，从浏览器打开可以正常显示页面，而在requests里面却被要求输入验证码或者重定向到其他的页面，后来明白了JS会发送鼠标点击事件，服务器收到被点击链接的请求，会看看之前是不是已经通过访问某个文件把信息发送过来了，如果发送过来了，服务器就认为是合法的浏览器，给出正常的网页内容，而requests没有鼠标事件响应，就直接访问了，所有服务器就拒绝服务了；
	还有一次是假链陷阱，就是解析列表页拿到详情页url，再访问详情页就可以获取到想要的数据，但是程序运行不一会就会出现问题，后来仔细排查发现目标网站在某些列表页中通过"display:none"的方式隐藏了一个<a>标签，页面显示是看不到的，但是页面源码中有，当我通过通用的规则解析出详情页url，访问到这个隐藏的url时，就会进入设置的陷阱；
	还有一些比较小的点，比如：
		1.请求头里边带“:”，这个东西是解析不了的，应该直接删除掉；
		2.使用pymysql连接数据库时，charset的内容是“utf8”，而不是“utf-8”；
		3.还有JS代码反格式化的问题；
```
